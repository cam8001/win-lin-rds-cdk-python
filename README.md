# Private Windows/Linux/SQL Infrastructure Stack

Python CDK stack deploying a secure, private infrastructure environment in **ap-southeast-6** (Asia Pacific - New Zealand).

## Architecture

```
VPC (10.55.0.0/16)
|
|-- Public Subnets (/24 x2 AZs)
|   |-- NAT Gateway (single)
|   |-- Internet Gateway
|
|-- Private Subnets (/20 x2 AZs)
|   |-- RHEL 9.6 Large (m7i.8xlarge) - Kubernetes/RKE2 host
|   |-- RHEL 9.6 Small (c7i.xlarge)
|   |-- Windows Server 2022 x2 (c7i.2xlarge)
|   |-- VPC Endpoints (SSM, SSM Messages, CloudWatch Logs)
|
|-- Isolated Subnets (/24 x2 AZs)
|   |-- RDS SQL Server 2022 Standard (db.m6i.xlarge)
|
|-- S3 Gateway Endpoint
```

## EC2 Instances

| Name | OS | Instance Type | vCPU | RAM | Disk | Purpose |
|------|----|---------------|------|-----|------|---------|
| LinuxLarge | RHEL 9.6 | m7i.8xlarge | 32 | 128 GiB | 2 TB gp3 | Kubernetes host (Docker, RKE2) |
| LinuxSmall | RHEL 9.6 | c7i.xlarge | 4 | 8 GiB | 400 GB gp3 | General purpose |
| Windows1 | Windows Server 2022 | c7i.2xlarge | 8 | 16 GiB | 120 GB gp3 | |
| Windows2 | Windows Server 2022 | c7i.2xlarge | 8 | 16 GiB | 120 GB gp3 | |

All EC2 EBS volumes are encrypted. Instances are in private subnets with outbound internet via NAT Gateway.

## RDS

- SQL Server 2022 Standard Edition, single node
- `db.m6i.xlarge` (4 vCPU, 16 GiB)
- Deployed in isolated subnets (no internet route)
- Storage encrypted, deletion protection enabled
- Admin credentials stored in AWS Secrets Manager (auto-generated by CDK)
- Accessible only from EC2 instances on port 1433


CDK and RDS will automatically create secret in AWS Secrets Manager with the
database endpoint, username, and password. You can retrieve that by browsing
to Secrets Manager in the console, or by running the below in aws cli:

`aws secretsmanager get-secret-value --secret-id rds-sql-server-credentials --region ap-southeast-6`

or (for more human readable output)

`aws secretsmanager get-secret-value --secret-id rds-sql-server-credentials --region ap-southeast-6 --query SecretString --output text | python3 -m json.tool`

## Networking

- **VPC CIDR:** 10.55.0.0/16
- **NAT Gateway:** Single (non-HA), in public subnet AZ1
- **VPC Endpoints:** SSM, SSM Messages, CloudWatch Logs (interface), S3 (gateway)
- **Security Groups:**
  - EC2: allows all traffic between instances in the group, all outbound
  - RDS: allows TCP 1433 from EC2 security group only, no outbound

## S3

- File drop bucket for EC2 instances to read/write files
- S3-managed encryption, SSL enforced, public access blocked
- EC2 instance role has read/write access

## IAM

- Shared EC2 instance role with:
  - `AmazonSSMManagedInstanceCore` (Systems Manager access)
  - Read/write to the S3 file drop bucket

## CloudFormation Outputs

The stack outputs instance IDs, private IPs for all EC2 instances, the RDS endpoint, and the S3 bucket name.

## Prerequisites

- AWS CDK CLI
- Python 3.14+ with virtual environment
- Valid AWS credentials with account access

## Deployment

```bash
# Activate virtual environment
source .venv/bin/activate

# Install dependencies
pip install -r requirements.txt

# Synth (requires valid credentials for AMI lookups on first run)
npx cdk synth

# Deploy
npx cdk deploy
```

## Regional Limitations (ap-southeast-6, March 2026)

- **Rocky Linux AMIs:** Official Rocky Linux Foundation AMIs (owner `792107900819`) are not published in this region. Using RHEL 9.6 (binary-compatible). To switch to Rocky, copy the official AMI from ap-southeast-2 or use a Marketplace reseller image.
- **EC2 Messages VPC Endpoint:** Not available in this region. SSM RunCommand/SendCommand routes via NAT Gateway. Session Manager is unaffected (uses `ssm` + `ssmmessages` endpoints).
- **WorkSpaces Secure Browser:** Not available. Requires deployment in ap-southeast-2 (Sydney).
- **Amazon Location Service:** Not available. Requires deployment in ap-southeast-2 (Sydney).
